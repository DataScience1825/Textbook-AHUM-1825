What is select()?

select()

If you review the R Documentation for the select() function you‚Äôll find the following explanation:

Select (and optionally rename) variables in a data frame, using a concise mini-language that makes it easy to refer to variables based on their name.

In practice, we will use select() to help make large datasets more manageable, focusing only on the variables, or columns, we want to work with. select() can also be helpful when displaying our datasets, so that we can highlight key variables and improve readability by renaming the often shortened keywords used in datasets (e.g.¬†avg_mpg -> Average Miles Per Gallon). Keep in mind, this is all done without modifying or altering the original dataset!

This is the most important part for using select(), the list of variables you want to keep or remove. This can be as simple as explicitly listing each variable you want to use or be a little more complex by using one of select()‚Äôs operators. The operators are defined below, along with code examples used on a simple dataset. 
simple <- tibble(columnA = 24, columnB = 80, columnC = -3, columnD = 11) 
simple

name

Explicitly name the variable(s) you want to keep and their order.

select(simple, columnC, columnA)

position

List the position of the variable(s) within the dataset that you want to keep and their new order (i.e.¬†1 for first, 2 for second, 5 for fifth)

select(simple, 2, 4, 1)

: (colon)

Select a range of variables to keep, using either their names or positions

select(simple, columnB:columnD)
select(simple, 1:3)

- (minus)

Choose a column to remove, using either its name or position

select(simple, -columnA)
select(simple, -3, -4)

Renaming Columns

select(data, new_variable_name = old_variable_name)

To rename a variable using select() all you have to do is provide a new variable name, however the syntax can be a little awkward since the new name comes first, before the equals sign (=).

Rename with One Word

The easiest option for renaming a column is to use a single word. Make sure when you are renaming your variables that they still follow good coding practices (lowercase, noun phrase, meaningful).

Meaningful Variable Names Meaningless Variable Names 
select(mpg, transmission = trans) %>%
  head(n=2)
 
select(mpg, t = trans) %>%
  head(n=2)

Rename with Multiple Words

Be careful when working with multiple words. At first you might try to write out a few words, but be careful variable names cannot contain spaces! Otherwise you‚Äôll get an error message like in the code chunk below. Instead make sure to separate your lowercase words with under_scores.

Meaningful Variable Names Meaningless Variable Names
Long winded variable names are unwieldy and prone to typos. Also never use the actual values for a variable as its name! If new values are added or removed, the name would no longer accurately describe the variable‚Äôs contents/purpose. 
select(mpg, drive_train = drv) %>%
  head(n=2)
 
select(mpg, front_rear_or_four_wheel_drive = trans) %>%
  head(n=2)

Be Careful of ‚Äù ‚Äù and .

When viewing code snippets online, you may see other styles for naming variables with multiple words. But be careful, while some are just different (e.g.¬†CamelCasing) others can be downright problematic. Specifically, don‚Äôt use dot.notation or ‚Äúquotation marks‚Äù.

Quotes ‚Äú or ‚Äô Dot . Notation
Double quotes may appear more readable, but it will force all future operations to use quotes as well. For example, if you pipe %>% the results from select to filter, you‚Äôll need to reference your renamed columns using quotes ‚Äù ‚Äú. In general, you should rename columns so that the quotes are removed.
 Using dots, i.e.¬†periods, to separate the words in your variable names used to be common practice in R and you can still find plenty of datasets that still follow Dot.Notation, like the iris dataset. This practice has fallen out of vogue because it resembles the same notation used by class methods in R. Classes are a topic beyond the scope of this tutorial, so to simplify just don‚Äôt use dot.notation. 
select(mpg, "city mpg" = cty) %>%
  filter("city mpg" > 24) %>%
  head(n=2)
 
select(mpg, Fuel.Type = fl) %>%
  head(n=2)

Helper Functions

Explicitly listing the variables you need can work for creating a quick subset but manually writing out all those variable names can get tedious real fast. If you need to make a complex selection of variables, turn to the helper functions!

contains()

The contains() function will check each variable name to see if contains a literal string. This can be handy to grab a subset of variables that share a common substring.

You can list a single substring

select(green_vehicles, contains("mpg")) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

or multiple substrings.

select(green_vehicles, contains(c("mpg", "stnd"))) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

starts_with()

The starts_with() function works similar to contains(), but is a bit more focused. Instead of searching the entire variable name for a given substring, it only checks at the start of the name. In other words, its checking for a given prefix.

You can list a single prefix

select(green_vehicles, starts_with("stnd")) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

or multiple prefixes.

select(green_vehicles, starts_with(c("stnd", "c"))) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

ends_with()

The end_with() function is the polar opposite of starts_with(). Instead of searching the for a given prefix, it checks the end of the name for given suffix.

You can list a single suffix

select(green_vehicles, ends_with("score")) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

or multiple suffixes

select(green_vehicles, ends_with(c("ion", "_mpg"))) %>% #Retrieve the miles per gallon variables
  head(2) #limit to only 2 rows

num_range()

num_range(‚Äúprefix‚Äù, start:end)

The num_range() function expands the functionality of starts_with(). It appends a given prefix with a series of numbers. This is a great way to search for columns in a generated dataset! The billboard dataset, packaged with tidyverse, lists the top 100 songs of year 2000 and provides each song‚Äôs rank for each week after it entered the top 100. Working with num_range() we can easily grab a specific subset of weekly rankings.

select(billboard, num_range("wk", 5:15)) %>% #Retrieve the rankings for week 5 to 15 
  head(2) #limit to only 2 rows

everything()

As has been repeatedly stated, writing out the names of all the remaining variables in a data frame can be so tedious! Luckily, there the everything() function. It will grab all of the columns not explicitly named and place them at the end select()‚Äôs new data frame. 

select(green_vehicles, model, year, smartway, everything()) %>% #Place all the remaining variables after smartway
  head(2) #limit to only 2 rows

dplyr::rename()

While select can be used for renaming columns, it does come with a big drawback : you need to list all of your columns to prevent them from being dropped or reordered. If you just need to rename a column, without creating a subset, use dplyr‚Äôs rename() function. Luckily, it has the same syntax as select() for changing names:

rename(data, new_variable_name = old_variable_name)
